# 伪 3D 卷轴动作游戏 - 开发计划

## 项目概述

- **游戏类型**: 3D 卷轴动作游戏 (类"别踩白块"4 通道 + 动作打击)
- **目标平台**: 移动端 (iOS/Android)
- **核心玩法**: 射击 (点击) / 斩击 (划动) 消灭下落敌人，分层推进机制

---

## 技术选型建议

### 推荐方案：Unity (C#)

**理由**:
1. 移动端 2.5D 游戏成熟解决方案
2. 完善的 UI 系统 (UGUI) 支持关卡编辑器开发
3. 对象池、物理检测等系统内置支持
4. 丰富的 Asset Store 资源可加速开发
5. 支持 IL2CPP 编译，性能优秀

## 开发阶段划分

### 阶段一：核心战斗原型 (Core Combat Prototype)

#### 1.1 基础框架搭建
- [ ] 创建 Unity 项目，配置 2D/2.5D 渲染管线
- [ ] 搭建基础场景结构 (Camera, Canvas, Game Manager)
- [ ] 实现 4 通道网格布局系统

#### 1.2 层级系统 (Layer System)
- [ ] 实现多层敌人容器管理
- [ ] 实现层级推进逻辑 (清空当前层 -> 下一层下落)
- [ ] 实现层级透明度控制 (当前层不透明，后续层半透明)
- [ ] 实现非当前层输入屏蔽

#### 1.3 敌人系统
- [ ] 创建敌人基类 (EnemyBase)
- [ ] 实现敌人类型：重装战士、轻装武士、BOSS 大名
- [ ] 实现弱点克制系统 (斩击/射击伤害倍率)
- [ ] 实现敌人占位检测 (1 格/2 格)

#### 1.4 玩家输入与判定
- [ ] 实现点击输入检测 (射击)
- [ ] 实现滑动输入检测 (斩击)
- [ ] 实现射击判定 (点状/小范围圆形)
- [ ] 实现斩击判定 (轨迹线性/多边形区域相交检测)
- [ ] 实现 Hitbox 碰撞检测系统

#### 1.5 战斗反馈
- [ ] 实现动态血条系统 (受伤显示，0.5 秒隐藏)
- [ ] 实现伤害数字特效
- [ ] 实现敌人死亡特效
- [ ] 实现对象池管理 (血条/特效)

#### 1.6 BOSS 机制
- [ ] 实现 BOSS QTE 系统 (随机间隔生成标志)
- [ ] 实现 QTE 判定 (○=斩击，—=射击)
- [ ] 实现 QTE 成功/失败反馈
- [ ] 实现 QTE 期间普通攻击兼容

#### 1.7 游戏流程
- [ ] 实现关卡限时系统
- [ ] 实现胜利条件检测 (清空所有层)
- [ ] 实现失败条件检测 (时间归零仍有存活敌人)
- [ ] 实现游戏结束 UI

```json
{
  "ID": "enemy_heavy_001",
  "Name": "重装战士",
  "BaseHP": 100.0,
  "LaneWidth": 1,
  "WeaknessType": "SLASH",
  "DamageMult_Slash": 2.0,
  "DamageMult_Shoot": 1.0,
  "Hitbox_Default": {
    "OffsetX": 0.1,
    "OffsetY": 0.1,
    "Width": 0.8,
    "Height": 0.8
  }
}
```

```json
{
  "LevelID": "LV_005",

  ]
}
```
